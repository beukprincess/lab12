Система контролю версій GIT.

Nov 12

Система керування файлами. GIT основи.

Що таке система контролю версій?

Система контролю версій (VCS, Version Control System) — це програмне забезпечення, яке допомагає розробникам відстежувати зміни у вихідному коді з часом. Це дозволяє повернутися до попередніх версій проєкту, відновити файли, переглянути історію змін і працювати над одним проєктом кільком людям одночасно, не заважаючи один одному.

Git — це розподілена система контролю версій, розроблена Лінусом Торвальдсом у 2005 році. Його ключова особливість полягає в тому, що кожен розробник має повну копію всього репозиторію, що робить роботу швидшою і надійнішою.

Основні можливості та переваги Git

  • Відстеження змін: Git дозволяє фіксувати зміни, що робляться в проєкті, і переглядати повну історію кожного файлу.
  • Співпраця: Кілька розробників можуть працювати над одним проєктом одночасно, об'єднуючи свої зміни без конфліктів.
  • Розгалуження (Branching): Створення незалежних гілок дозволяє експериментувати з новими функціями, не впливаючи на основну робочу версію коду. Це одна з найпотужніших можливостей Git.
  • Відновлення: Ви завжди можете повернутися до попереднього стану проєкту, скасувати зміни або відновити видалені файли.
  • Швидкість та надійність: Завдяки розподіленій архітектурі, більшість операцій виконуються локально, що значно прискорює роботу.

10 базових команд Git

Ось список основних команд, які необхідно знати для початку роботи з Git:


# 1. Створює новий локальний репозиторій Git у поточній папці
git init

# 2. Клонує віддалений репозиторій на ваш локальний комп'ютер
git clone https://ru.wikipedia.org/wiki/%D0%A0%D0%B5%D0%BF%D0%BE%D0%B7%D0%B8%D1%82%D0%BE%D1%80%D0%B8%D0%B9

# 3. Додає файл або папку до "зони підготовки" (staging area)
# Додає конкретний файл
git add [ім'я файлу]
# Додає всі змінені файли
git add .

# 4. Фіксує додані зміни з описом
git commit -m "Коментар до змін"

# 5. Показує стан робочої директорії та зони підготовки
git status

# 6. Показує історію комітів
git log

# 7. Створює нову гілку та переходить на неї
git checkout -b [ім'я гілки]

# 8. Перемикається на існуючу гілку
git checkout [ім'я гілки]

# 9. Завантажує останні зміни з віддаленого репозиторію
git pull

# 10. Відправляє локальні коміти на віддалений репозиторій
git push
